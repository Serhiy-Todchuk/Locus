## Implementation notes

- **gumbo replaces the regex HTML stripper.** [src/extractors/html_extractor.cpp](../../src/extractors/html_extractor.cpp) now walks the real DOM and drops `script` / `style` / `noscript` / `nav` / `footer` / `header` / `aside` / `template` subtrees, emitting a block newline after `p`/`div`/`li`/`h1-6`/etc so paragraph + heading boundaries survive. Headings record gumbo's source line. Both `extract()` and `extract_from_string()` (used by ZIM + web-fetch) route through the same walk, so ZIM article extraction inherits the improvement for free. The old S2.3 entity/whitespace helpers are gone; gumbo decodes entities itself.
- **Web content is its OWN store, not the workspace index.** [src/web/web_cache.{h,cpp}](../../src/web/web_cache.h) owns `web_pages` / `web_fts` / `web_headings` in the main `index.db` (lazily created on first WebCache construction). The taint columns (`origin="web"`, `injection_flags`) live on `web_pages`; the search read path JOINs them. Eviction is TTL (`cache_ttl_hours`) + size cap (`cache_max_mb`), both run on workspace open. `WebCache` is created only when `capabilities.web_retrieval` is on, never in ZIM mode.
- **Provider client is decoupled from the LLM transport.** [src/web/web_search_provider.{h,cpp}](../../src/web/web_search_provider.h) is a thin cpr wrapper: `web_search_query` (Brave via `X-Subscription-Token`, SearXNG via `?format=json`) and `web_fetch_url` (plain GET, HTTPS-only unless `allow_http`, identifies as `Locus/1.0`). The JSON adapters (`parse_brave_results` / `parse_searxng_results`) are pure + unit-tested.
- **"Index, don't inject."** `web_fetch` returns ONLY the page outline (~30 tokens); the full body is stored, never returned. The agent then pulls sections via `web_read(url, heading=...)` or finds terms via `search_text(query, sources="web")`. Fetched bodies go through the S6.0 `scan_for_injection` + `apply_injection_policy` hook before storage; `web_read` re-stamps the taint marker at the read site via `render_tainted_snippet`.
- **Approval.** `web_search` / `web_fetch` default to `ask` (the user sees the query / URL before anything leaves the machine -- the real control per the threat model); `web_read` is `auto_approve` (local, already-scanned cache).
- **Settings GUI** (follow-up, same milestone). Added a `Settings > Web` tab ([src/frontends/gui/settings/web_settings_panel.{h,cpp}](../../src/frontends/gui/settings/web_settings_panel.h)) exposing all nine `WorkspaceConfig::Web` knobs (api_key is a password field). The same pass added the `Settings > Security` tab for the S6.0 scanner knobs (`injection_scan` / `scan_zim` / `block_confidence` / `max_scan_kb`) which were previously file-only. `Workspace::sync_web_cache()` lets enabling web retrieval take effect without a workspace reopen (creates/drops the `WebCache` + re-runs eviction when the Settings change touches web or capabilities). UIA: [security_settings.json](../../tests/ui_automation/scripts/security_settings.json) + [web_settings.json](../../tests/ui_automation/scripts/web_settings.json) -- widget reachability + a `.locus/config.json` round-trip assertion; both pass.

See [architecture/web-retrieval.md](../../architecture/web-retrieval.md) for the detailed pipeline (fetch -> index -> search), token budget strategy, and session-scoping rationale.
