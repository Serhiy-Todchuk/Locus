# S6.2 -- ZIM Reader (Wikipedia / Kiwix)

## Context

[S5.N](../M5/S5.N-non-code-workspace-proof.md) proves the non-code
workspace story on a *pre-extracted* ZIM via
[scripts/build_ws2.ps1](../../scripts/build_ws2.ps1) (downloads a Simple
English Wikipedia ZIM, uses `zimdump.exe` to dump articles into
`A/*.html`, indexes that flat folder). That gets us a working Wikipedia
demo today without `libzim`, at the cost of ~3x disk and a one-time
extraction step. S6.2 replaces the extraction step with live
`zim::Archive` access -- the same retrieval + agent loop, but the
"files" are virtual entries inside the ZIM rather than real HTML on
disk. The WS2 acceptance prompts from S5.N should keep passing
unchanged once S6.2 lands; only the file backend changes.

## Tasks (done 2026-06-27)

Note: the dependency decision changed during implementation. libzim is
GPL-2.0-or-later and would relicense the MIT Locus binary, so instead of adding
it we wrote a focused MIT ZIM v6 reader ([src/zim/zim_archive.{h,cpp}](../../src/zim/zim_archive.h))
against the documented format, with permissive `zstd` + `liblzma` for cluster
decompression. The `ZimWorkspace` design also folded into `Workspace` as a
read-only "ZIM mode" rather than a separate class, so the existing
`LocusSession` / tools / agent / GUI need no interface change.

- [x] MIT ZIM reader opens a `.zim` file (mmap; header + dirents + clusters; zstd/xz decompress). [no libzim -- see note]
- [x] `Workspace` ZIM mode wraps a `.zim` as a read-only virtual workspace; articles are virtual "files"
- [x] Article iterator feeds indexer: title -> path, HTML content -> stripped plain text (`Indexer::index_virtual_article`)
- [x] `list_directory` enumerates ZIM articles from the index (no real FS to walk)
- [x] `read_file` returns stripped article text (served from the index for virtual paths)
- [x] Index build progress for large ZIM (per-500-article `on_progress`; bulk insert under one transaction)
- [x] Catch2 tests: open the real ready.gov test ZIM, verify article retrieval + FTS search (skip when absent)
- [x] Untrusted-origin tagging ([S6.0](S6.0-prompt-injection-scanner.md)): rows tagged `origin="zim"`; search read path stamps `[wikipedia, untrusted]`. Origin stamp REQUIRED; per-chunk keyword scan opt-in via `security.scan_zim` (default false). [S6.0 Task D]

## Notes

`ZimWorkspace` is a virtual workspace -- read-only, no file watcher, no chunking pipeline beyond the initial bulk build.

The "trusted workspace vs untrusted external ingress" boundary is in [architecture/overview.md](../../architecture/overview.md) `## Trust Boundary`. A ZIM dump is untrusted even though it is a local file: the user chose the source (a Kiwix snapshot), not the content of millions of public-authored articles.
