# S6.19 -- Tool-Arg Type Coercion + Transient-Endpoint Retry

**Done 2026-06-06.** Both tasks shipped in one pass. Surfaced by an agentic-mode
session driving `qwen/qwen3-coder-480b-a35b-instruct` on the NVIDIA Build hosted
endpoint (the DX12 + DXR RTX-GI app build, output under
[tests/ui_automation/output/agentic_TestQwenVibeGI/findings.md](../../tests/ui_automation/output/agentic_TestQwenVibeGI/findings.md)).

## Problem

Two distinct robustness holes, both exposed the moment a real hosted model drove Locus
through a long multi-round build:

### A. A malformed tool-arg TYPE crashed the whole GUI

`qwen3-coder` emitted tool args with the WRONG JSON type -- Python-style string
booleans (`"overwrite":"True"`), a JSON array double-encoded as a string
(`"edits":"[{...}]"`), and integers as strings (`"offset":"2"`). Locus read these with
`nlohmann::json::value<T>(key, default)` / `.get<T>()`, which **throw** `type_error` on a
type mismatch. The throw landed in two places:

- **`ITool::preview()`**, which runs on the agent thread during the
  `on_tool_call_pending` broadcast (even for auto-approved tools, to render the bubble).
  That call site had no try/catch, so the throw escaped the agent thread to
  `std::terminate` -> Windows fail-fast (`0xc0000409`). **`locus_gui.exe` crashed
  outright** -- a single mistyped token from the model killed the app. Hit 3 times in one
  session (bool, array, int).
- **`execute()`** for the int case -- there the dispatcher catch-all caught it, but it
  surfaced as an opaque "tool failed with an internal error" result. The model couldn't
  tell it had mistyped the arg, re-sent the identical call, and the QualityMonitor
  repeated-tool-call cap aborted the turn after 2 nudges. Wasted a whole turn.

### B. A transient endpoint failure killed the whole turn with no retry

The free NVIDIA tier intermittently returned HTTP 429 (rate-limited on rapid calls),
HTTP 5xx, and -- the nastiest -- a clean **HTTP 200 with an empty body / zero SSE chunks**
after a multi-minute queue wait. `OpenAiTransport` only retried on
`cpr OPERATION_TIMEDOUT` (once). 429 / 5xx / empty-200 all fell straight through to
`on_error`, which aborted the turn and discarded all progress. On a flaky hosted endpoint
this made long agentic sessions effectively impossible -- any one transient blip mid-build
threw away 15-40 minutes of work.

## Fix

### A. Never-throwing tool-arg coercion + a preview() backstop

Three tolerant readers in [src/tools/shared.{h,cpp}](../../src/tools/shared.h), each of
which **never throws** -- on a type it can't use it returns the caller's fallback:

- `coerce_bool(args, key, fallback)` -- accepts a real bool, 0/1 ints, and the strings
  `true/false/1/0/yes/no/on/off` (case-insensitive).
- `coerce_json_array(args, key)` -- returns a real array as-is, OR JSON-parses a string
  that encodes an array (the `"edits":"[...]"` case), else an empty array. `edit_file`
  now *applies* the stringified-array form instead of erroring on it.
- `coerce_int(args, key, fallback)` -- accepts int / float (truncates) / numeric string
  (`"100"`, `" 100 "`, `"-5"`, even `"100px"`) / bool.

Routed through every LLM-reachable arg site: `write_file`/`edit_file` `overwrite`,
`edit_file` `replace_all` + `edits`/`edits_array`, `add_memory` `pinned`, `search_*`
`case_sensitive` + all five `max_results`, `read_file` `offset`/`length`,
`list_directory` `depth`/`max_entries`, `run_command` `timeout_ms`, process tools
`process_id`/`since_offset`. Both preview() AND execute() paths.

**The backstop that closes the whole crash class:** `tool->preview(call)` in
[ToolDispatcher](../../src/agent/tool_dispatcher.cpp) is now wrapped in try/catch (std +
`...`), logging a warn and using a fallback preview string. So even a malformed arg this
pass didn't anticipate can no longer escape the agent thread and fail-fast the process --
preview() now has the same exception guarantee execute() always had.

### B. Bounded transient-failure retry with backoff

`classify_retry(...)` in [src/llm/openai_transport.{h,cpp}](../../src/llm/openai_transport.h)
-- a pure, unit-tested decision over one attempt's outcome:

- **Retryable (transient):** `OPERATION_TIMEDOUT`, HTTP 429, HTTP 5xx, and HTTP 200 with
  zero chunks (empty body).
- **Not retryable (deterministic):** connect-refused, 4xx other than 429 (bad request /
  auth), and any cancelled call.
- Exponential backoff off a base, capped, with the floor raised to the server's
  `Retry-After` when present. **429s get a higher floor (8s)** than other transients --
  rate-limit windows on free tiers outlast a 2s/4s backoff (observed exhausting 3 quick
  retries against a sustained 429 in the wild).

`do_post` is now a bounded retry loop (`k_max_attempts = 5`) that re-checks the cancel
flag across each backoff. Live-validated: empty-200 and 429 both retried transparently
mid-build where they previously aborted the turn.

## Key invariants added

- **Tool-arg reads on the model path go through `coerce_*` or are `.is_*()`-guarded; no
  raw `.value<T>` / `.get<T>()` on `call.args` may throw to a caller.** preview() has a
  dispatcher-level try/catch backstop regardless. (See CLAUDE.md Key invariants.)
- **Transient transport failures retry; the retry decision is the single pure function
  `classify_retry`.** `do_post`'s loop only does side effects (sleep, cancel re-check,
  recurse). Deterministic failures never retry.

## Not done (follow-ups -- the agentic-session "Key product finding")

The session also surfaced a larger agent-productivity gap that is NOT addressed here
(documented in findings.md #6, kept for a future stage):

- A **"stuck on the same compile error" detector** (hash the normalized error signature
  across rounds; nudge/escalate after K repeats) -- the agent thrashed 38-72 rounds on
  large files reintroducing the same errors.
- A **soft nudge toward targeted edits over whole-file rewrites** for large files.
- **Type-aware tool-error messages** (name the arg + expected type) for the residual
  cases coercion doesn't cover.
- A **first-class workspace-dependency declaration** (the agent only learned a header was
  missing by failing to compile, repeatedly).
- **Surfacing retry/round state in the chat footer** so a watching human sees thrash.
