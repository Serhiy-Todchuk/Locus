# S6.13 -- Reasoning Watchdog + "Commit now" Button

## Problem

Reasoning-heavy local models (Qwen 3.x with thinking on, Qwen3.6-35b-a3b, DeepSeek-R1 distills) can spiral inside a single LLM round and never emit a tool call. Two real cases from agentic-mode QA:

1. 2026-05-24 TestLocalVibe run, Task 2 round 3-4 on `qwen/qwen3.6-27b`: 25+ min of streaming reasoning content (666 KB / 2500+ chunks) with no tool call before the harness watchdog had to manually abort. Same prompt with `/no_think` finished in 183 s.
2. 2026-05-19 agentic Tetris run 1 round 10 on `qwen/qwen3.6-35b-a3b`: 12 602 chars of reasoning in ~10 min while the model thrashed on a SHAPE-array bounds error.

Today the only escape is the Stop button, which cancels the whole turn and discards the partial output. Cancellation latency was measured at 7+ minutes between click and the `Turn cancelled` log line on the TestLocalVibe run -- the model has to roll its own state to a chunk boundary first.

[S6.14](S6.14-thinking-knob.md) addresses the cause (disable thinking at source). This stage addresses the symptom (cap runaway rounds even when thinking is intentionally on, e.g. for genuinely-hard tasks where it helps for a while then breaks).

## Lands in M6 because

- Independent of S6.11/S6.12 (prompt-time) and S6.14 (request-time); this is stream-time.
- Surfaces involved are all already present: [`AgentLoop`](../../src/agent/agent_loop.cpp) sees the stream events, [`AgentCore`](../../src/agent/agent_core.cpp) owns round state, [`ChatPanel`](../../src/frontends/gui/chat_panel.cpp) renders the chat-footer action row.

## Design

### Triggers (OR semantics)

Three new `WorkspaceConfig::Agent` fields, all default to `0 = off`:

- `reasoning_max_seconds` -- wall-clock seconds since the current round started AND no tool call has been emitted.
- `reasoning_max_chars` -- accumulated `on_reasoning_token` chars in the current round.
- `reasoning_max_rounds_silent` -- consecutive LLM rounds since the most recent tool call within the current user turn.

Any non-zero threshold that is exceeded fires the watchdog. The check runs once per stream chunk in `AgentLoop`'s callback path (cheap -- one int comparison per channel per chunk).

### Action

Two behaviour modes via `WorkspaceConfig::Agent::reasoning_auto_nudge` (default `false`):

- **Manual mode (`auto_nudge = false`):** when a threshold trips, a non-modal "Commit now" button surfaces in the chat-footer action row (beside Stop). The button is named `locus.chat.commit_btn` and is hidden whenever the watchdog has not tripped on the current round. User clicks -> watchdog fires the action.
- **Auto mode (`auto_nudge = true`):** the watchdog fires the action immediately on trip. Designed for agentic harness driving where no user is present to click the button.

The action itself (identical in both modes):

1. `AgentCore::request_commit_now()`:
   - Signal the in-flight LLM stream via the existing `cancel_flag_`. Wait for `LMStudioClient` to land cleanly (same mechanism the Stop button uses, but the turn is NOT cancelled).
   - Compose a synthetic user-style message: `"[Auto-nudge] Stop reasoning and either commit to a tool call now, or give a brief final answer. Do not produce more reasoning."` Append to history as a normal user message (so the model sees it next round; the agent thread is the writer per S3.I / S5.J).
   - Increment the per-turn nudge counter. Hard-cap at 2 nudges per user turn; if a third would fire, abort the whole turn with `"Agent appears stuck (3 reasoning watchdog trips in one turn). Stopping."`
   - Reset the reasoning-channel counters and start a new round.
2. Activity log gets a `reasoning_watchdog { trigger: "<which>", value: <int>, threshold: <int>, nudge_count: <n> }` event for transparency.
3. Chat-footer chip `nudges: N` appears when N > 0; clicking filters Activity log to watchdog events.

### Chat panel UI

- New button `locus.chat.commit_btn` in the existing action row that already hosts Stop / Submit / Compact / Auto-compact / Undo / Find / preset_choice / mode tri-state.
- Label: "Commit now". Tooltip: "Cancel the current LLM round and ask the model to commit to a tool call or final answer."
- Visible only when watchdog has tripped on the current round AND `agent_busy = true`. Hidden when agent goes idle or model emits a tool call (counter reset).
- `IFrontend` gains `on_reasoning_watchdog_tripped(round_index, trigger)` and `on_reasoning_watchdog_cleared()` events; `ChatPanel` listens and toggles button visibility.

### State machine

```
+--------+  threshold trip   +-----------+
| ARMED  +------------------>| TRIPPED   |
| (round +<--+               | (button   |
| start) |   | tool call     |  visible) |
+---+----+   +---------------+-----+-----+
    ^                              |
    | new round                    | click | auto_nudge fires
    |                              v
    |                        +-----+-----+
    +------------------------+ NUDGING   |
                             | (stream   |
                             |  cancel,  |
                             |  msg add) |
                             +-----------+
```

`ARMED` -> `TRIPPED`: counter exceeds any threshold.
`TRIPPED` -> `ARMED`: a tool call is dispatched (round counter resets), or the round ends naturally (model committed without prompting).
`TRIPPED` -> `NUDGING`: user clicks the button OR `auto_nudge=true` AND threshold trips.
`NUDGING` -> `ARMED`: new round starts; counters cleared.

Hard-cap of 2 nudges per turn lives in `AgentCore`; on the 3rd would-fire, abort with the message above.

### Counter semantics

- `seconds`: a `std::chrono::steady_clock::time_point` captured at round start, polled in the chunk callback. Cheap.
- `chars`: an `int` accumulated on each `on_reasoning_token`. Resets on round boundary.
- `rounds_silent`: tracked on `AgentCore` across the turn; bumped at round start, reset when ANY tool dispatch succeeds.

All three are independently checked. The first to trip wins (logged: which trigger fired).

## Notes

- The 2-nudge cap is deliberate. A model that's actually broken (not just slow-to-commit) will not produce useful output on the 3rd retry; better to surface the failure than keep nudging.
- The nudge message is hardcoded for v1 -- user-configurable wording is in scope for a future stage if telemetry shows the default wording is suboptimal.
- The watchdog does NOT distinguish "reasoning round" from "text response round" today. A model that streams plain text content without ever calling a tool could also trip the char threshold. That's mostly fine -- if the model is producing thousands of chars of text without committing to anything, a commit-now nudge is reasonable. Document this behaviour; do not over-engineer the trigger.
- Composes cleanly with [S6.14](S6.14-thinking-knob.md): with thinking off and the watchdog set tight, even a Qwen model with broken stop-conditions can't run away.
