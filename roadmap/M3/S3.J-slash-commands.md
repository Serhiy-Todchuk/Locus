# S3.J — Extract slash-command parser + dispatcher

## Problem

[src/agent_core.cpp](../../src/agent_core.cpp):683-846 holds `try_slash_command()` — a 160-line ad-hoc shell-style tokenizer with quote handling, key=value parsing, type coercion, positional/named arg mapping, `/help` rendering, and tool execution + result formatting. It lives inside `AgentCore` for no structural reason — it never touches the LLM, history, or activity log. It is pure user-input → tool-call translation.

The parser itself is tested only by hand. It has known sharp edges (no escape sequences, no `--flag` style, type coercion errors throw `std::stoi`, no completion hooks). M4 will want:

- **S4.G (MCP)**: dynamic tools registered at runtime — `/help` should group them by server.
- **GUI slash-command popup** (already partially landed in [src/gui/chat_panel.cpp](../../src/gui/chat_panel.cpp)): needs the same tokenizer for autocomplete, currently re-implements a half-version.
- **S4.D (plan mode)**: slash commands like `/plan` should switch agent state, not invoke a tool.

Leaving this trapped inside `AgentCore` means every one of those callers either re-implements the parser or pulls in the entire agent.

## Target

```
src/agent/slash_commands.{h,cpp}

class SlashCommandParser {
public:
    struct ParsedCall {
        std::string tool_name;
        nlohmann::json args;
        bool         is_help = false;
    };
    // Returns nullopt if `input` is not a slash command (no leading '/').
    // Throws SlashParseError on malformed input (unterminated quote, bad type).
    static std::optional<ParsedCall> parse(std::string_view input,
                                           const IToolRegistry& tools);
};

class SlashCommandDispatcher {
public:
    SlashCommandDispatcher(IToolRegistry& tools, IWorkspaceServices& services);

    // Returns true if input was a slash command (handled or rejected with error).
    // `on_output` receives display text (markdown).
    bool try_dispatch(std::string_view input,
                      const std::function<void(std::string)>& on_output,
                      const std::function<void(std::string)>& on_error);

    // For autocomplete: returns matching tool names + param hints.
    std::vector<CompletionItem> complete(std::string_view prefix) const;
};
```

`AgentCore::try_slash_command` becomes:

```cpp
return slash_dispatcher_.try_dispatch(
    content,
    [&](std::string s) { frontends_.broadcast([&](IFrontend& fe) { fe.on_token(s); }); },
    [&](std::string s) { frontends_.broadcast([&](IFrontend& fe) { fe.on_error(s); }); });
```

## Implementation notes

- `SlashCommandParser::parse()` is a pure static function; `SlashCommandDispatcher` is the only stateful piece and it just holds refs to `IToolRegistry` + `IWorkspaceServices` (no heap members, no threads). That keeps future reuse from an MCP-populated registry (S4.G) or a plan-mode gate (S4.D) trivial.
- Parser throws `SlashParseError`; dispatcher is the only translation boundary to user-visible text. `AgentCore` never sees the exception type.
- `/help` stays in the dispatcher because it needs the live tool list with param signatures; the parser just signals `is_help = true`.
- Boolean coercion now accepts `true/1/yes/on` and `false/0/no/off` (case-insensitive) — strictly more permissive than the old `"true" || "1"`, but backward-compatible.

## Notes

- Parser is pure function; dispatcher takes `IWorkspaceServices` (S3.C) for tool execution. Order: do **after** S3.C so `IWorkspaceServices` exists.
- Don't introduce a new "command" concept that's not a tool. `/plan` (M4 S4.D) should be implemented as a tool that flips plan-mode state, not as a special-cased slash. Keeps the parser simple.
- `/help` formatting is the dispatcher's job, not the parser's — parser just returns `is_help = true`.
