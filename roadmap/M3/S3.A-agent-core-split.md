# S3.A — Split `AgentCore`

## Problem

[src/agent_core.cpp](../../src/agent_core.cpp) is 850 LOC with ~14 responsibilities: LLM loop, tool routing, approval gating, slash-command parsing (~160 LOC), activity ring buffer, session I/O, token accounting, context overflow, compaction, cancellation, sync/async duality, queue pump, system-prompt seeding, frontend fan-out wiring.

M4 piles on: plan mode (S4.D), checkpoint/undo (S4.B), verify loop (S4.C), parallel tool calls (S4.H), subagents (S4.U), tool-format parsers per model family (S4.N), weak/strong router (S4.Q), background processes (S4.I), memory bank (S4.R), file-change awareness (S4.T). Each wants to hook somewhere in `run_llm_step` or `process_tool_call`. If those two methods keep growing, every M4 stage becomes a merge conflict with itself.

## Target structure

```
src/agent/
  agent_core.{h,cpp}           orchestration only: thread, queue, public API, frontend reg
  agent_loop.{h,cpp}           run_llm_step + round-limiter (one turn)
  tool_dispatcher.{h,cpp}      approval gate + execute + result injection + policy resolve
  activity_log.{h,cpp}         ring buffer + emit + late-joiner API
  context_budget.{h,cpp}       token accounting + compaction policy
  session_store.{h,cpp}        promote SessionManager; prepare for checkpoint hook (S4.B)
```

`AgentCore` becomes a ~150-line coordinator that owns these as members. Public `ILocusCore` API is unchanged.

## Notes

- Do **not** change `IFrontend` or `ILocusCore` public interfaces. This is pure internal restructuring.
- Threading: currently only the agent thread touches `history_`. Preserve that invariant — none of the new collaborators should hold their own mutex on conversation history.
- Dependency injection via references (not pointers) where lifetimes are clearly nested inside `AgentCore`. Only `IFrontend*` fan-out stays pointer-based (already is).
