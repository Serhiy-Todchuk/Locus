# S3.E — Split `tools.cpp` into `src/tools/`

## Problem

[src/tools.cpp](../../src/tools.cpp) is 21 KB with 12 tool implementations in one file. M4 adds at least 20 more:

- S4.A: `edit_file`, `multi_edit`, `apply_patch`
- S4.P: `grep`
- S4.M: `ast_search`
- S4.L: `git_status`, `git_diff`, `git_log`, `git_blame`
- S4.E: `goto_definition`, `find_references`, `get_diagnostics`, `rename_symbol`, `hover_info`
- S4.I: `run_command_bg`, `read_process_output`, `stop_process`, `list_processes`
- S4.U: `spawn_subagent`
- S4.R: `session_notes`
- S4.D: `propose_plan`

That's a 60+ KB single file. AI collaborators and humans both struggle with files that long. Merge conflicts compound when multiple stages land in parallel.

## Target structure

```
src/tools/
  tools.h                      aggregator header: register_builtin_tools(registry)
  tools.cpp                    one-liner registration factory
  shared.{h,cpp}               resolve_path, error_result, make_relative helpers
  file_tools.{h,cpp}           read/write/create/delete  + M4: edit/multi_edit/apply_patch
  search_tools.{h,cpp}         search_text/search_symbols/search_semantic/search_hybrid
                               + M4: grep, ast_search
  index_tools.{h,cpp}          list_directory, get_file_outline
  process_tools.{h,cpp}        run_command  + M4: run_command_bg, read_process_output, stop_process, list_processes
  interactive_tools.{h,cpp}    ask_user  + M4: session_notes, propose_plan
  git_tools.{h,cpp}            (empty in M3; M4 S4.L fills)
  lsp_tools.{h,cpp}            (empty in M3; M4 S4.E fills)
  mcp_tools.{h,cpp}            (dynamic proxies; M4 S4.G)
```
