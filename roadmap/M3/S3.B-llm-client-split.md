# S3.B — Split `ILLMClient`

## Problem

[src/llm_client.cpp](../../src/llm_client.cpp):286–506 has a single 220-line `do_stream` function that hard-codes: OpenAI JSON wire format, single-endpoint transport, inline SSE delta parsing, inline tool-call accumulation, inline reasoning-channel split, inline usage parsing, cpr retry logic.

M4 breaks every assumption:
- **S4.N** — Qwen `<tool_call>`, Claude-style `<function_calls>`, Llama3 plain-JSON.
- **S4.O** — partial streaming tool output.
- **S4.Q** — weak + strong model split (two endpoints, one agent).
- **S4.F** — prompt-cache discipline (byte-stable prefix audit).
- **S4.H** — parallel tool-call parsing (indexed deltas — already works).

`estimate_tokens` as a static-on-interface is a code smell; it also blocks S4.F.

## Target structure

```
src/llm/
  llm_client.h                 ILLMClient interface only
  openai_transport.{h,cpp}     cpr + SSE; emits raw delta events
  sse_parser.{h,cpp}           (exists; fine)
  stream_decoder.h             IStreamDecoder interface (parses deltas)
  stream_decoders/
    openai_decoder.{h,cpp}     current behavior
    qwen_xml_decoder.{h,cpp}   S4.N
    claude_xml_decoder.{h,cpp} S4.N
  llm_router.{h,cpp}           Role::Strong / Role::Weak client selection (S4.Q hook, ship empty)
  token_counter.{h,cpp}        heuristic today; swappable later
```
