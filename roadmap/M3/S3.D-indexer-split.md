# S3.D ŌĆö Split `Indexer`

## Problem

[src/indexer.cpp](../../src/indexer.cpp) is 700 LOC / 20 methods ŌĆö roughly as big as `AgentCore`. It mixes: Tree-sitter parser lifecycle, languageŌåÆgrammar map, per-language `SymbolRule` tables, symbol-name recursion, file I/O, FTS/symbols/headings/chunks SQL persistence via 12 raw `sqlite3_stmt*` pointers, chunking dispatch, incremental updates, extension-based language detection.

The `ExtractorRegistry` pattern already proved the model works: adding docx/xlsx/pdf did **not** touch `indexer.cpp`. Apply the same to the Tree-sitter side.

M4 pressure points:
- **S4.E (LSP)** will want the language-grammar map externally accessible for diagnostics + the extractor-registry shape.
- **S4.L (git-native)** wants a `.gitignore` hook into indexing.
- **S4.M (ast_search)** wants to *reuse* the languageŌåÆgrammar map.
- **S4.J (better embeddings + reranker)** wants to re-embed with a different model; Indexer and Embedder are currently coupled through `on_chunks_created`.

## Target structure

```
src/index/
  indexer.{h,cpp}              orchestration; NO tree-sitter includes
  symbol_extractor.h           ISymbolExtractor interface
  symbol_extractors/
    cpp_extractor.cpp          per-language rules
    python_extractor.cpp
    js_ts_extractor.cpp
    rust_extractor.cpp
    go_extractor.cpp
    java_extractor.cpp
    csharp_extractor.cpp
  tree_sitter_registry.{h,cpp} lang ŌåÆ TSLanguage*; shared with S4.M ast_search
  prepared_statements.{h,cpp}  RAII holder for the 12 sqlite3_stmt*
  index_query.{h,cpp}          (exists; leave)
  chunker.{h,cpp}              (exists; leave)
  glob_match.h                 (exists; leave)
```

## Tasks

- [ ] Create `src/index/` subfolder; move existing `indexer.{h,cpp}`, `index_query.{h,cpp}`, `chunker.{h,cpp}`, `glob_match.h` into it.
- [ ] Extract `TreeSitterRegistry`: owns `TSParser*`, the `ts_languages_` map, and `language_for(ext)` / `language_for_name(name)`. `Indexer` holds one reference.
- [ ] Extract `ISymbolExtractor` with `extract(TSNode root, string_view source) -> vector<SymbolSpan>`. Move the `SymbolRule` tables + `extract_name_from_node` into per-language `.cpp` files that each register via a static registrar (mirror `ExtractorRegistry`).
- [ ] Extract `PreparedStatements` RAII holder: ctor prepares all 12 statements, dtor finalises. `Indexer` destructor becomes a one-liner.
- [ ] `Indexer::index_file` shrinks: read file ŌåÆ pick extractor ŌåÆ upsert ŌåÆ call `SymbolExtractor` ŌåÆ chunk ŌåÆ persist. The per-language logic leaves the file.

## Notes

- Do not change the DB schema. This is code organization only.
- `TreeSitterRegistry` is deliberately shared with S4.M (ast_search) ŌĆö shipping it as a reusable component now means S4.M gets a free head-start.
- Keep symbol-rule tables as `static const std::vector<SymbolRule>` in each per-language `.cpp` ŌĆö no runtime overhead, full visibility in source.
