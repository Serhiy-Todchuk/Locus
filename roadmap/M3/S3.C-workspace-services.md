# S3.C — `IWorkspaceServices` interface ✔

## Problem

[src/tool.h](../../src/tool.h):70 — `WorkspaceContext` is a raw struct with 4 nullable pointers today (`root`, `index`, `embedder`, `workspace`). By end of M4 it needs, at minimum: `CheckpointStore*`, `LspClient*`, `ProcessRegistry*`, `McpRegistry*`, `GitClient*`, `VerifyRunner*`, `SubagentPool*`, `SessionNotes*`, `ActivityLog*`. That's ~12 nullable raw pointers every tool has to null-check — a classic god-struct anti-pattern, and every new subsystem triggers a struct edit + every tool that uses it needs updating.

## Notes

- Reference-based (`IWorkspaceServices&`) where the tool is guaranteed non-null; pointer-based only inside the interface for optional subsystems.
- This unblocks tool-level testing significantly — `FakeWorkspaceServices` is ~20 lines vs today's `WorkspaceContext{...}` which still needs real `IndexQuery`.
- Do **not** move `Workspace` itself into `src/core/` in this stage — that's S3.H. Just add the services interface.

## Implementation notes (post-merge)

- `index()` ended up returning `IndexQuery*` (nullable) instead of the sketch's `IndexQuery&`. The fake-services test ergonomics dominated: the file-op tool tests (read/write/create/delete) never need an index and would otherwise have to spin up a real `Workspace` + SQLite DB just to satisfy a reference. Tools that use the index keep their single-line null check.
- `Workspace` inherits `IWorkspaceServices` directly. No adapter class; `workspace()` returns `this` so the approval-policy override path in `AgentCore` keeps working.
- The dead `Embedder* Workspace::embedder()` accessor was removed — it conflicted with the new `EmbeddingWorker* embedder()` override and had zero callers.
- The `ws.embedder` + `ws.workspace->embedding_worker()` fallback in the semantic/hybrid tools collapsed into a single `ws.embedder()` call — the override already reads the live `embedding_worker_` pointer, so hot-toggle is preserved without the fallback.
