# S4.Q — Multi-Model Split (Weak + Strong)

## Problem

Aider's `--weak-model` uses a cheap model for summarization/naming, strong model for reasoning. On local GPUs (tight VRAM), this is the difference between feasible and infeasible.

## Tasks

- [ ] `llm.weak_endpoint` + `llm.weak_model` config (fallback to main if unset)
- [ ] Weak-model use cases: compaction summary, session titling, commit message generation, activity-event summaries
- [ ] `ILLMClient` factory per role: `get_client(Role::Strong | Role::Weak)`
- [ ] UI: settings lists both; allow same endpoint with different model names
