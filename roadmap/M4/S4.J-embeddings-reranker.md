# S4.J — Better Embeddings + Reranker

## Problem

`all-MiniLM-L6-v2` is 2019-era, 256-token, English-only, general-domain. For code identifiers, non-English docs, long headings — recall is mediocre. No reranker means top-3 precision is bi-encoder-quality only. Every production RAG stack reranks.

## Tasks

- [ ] Evaluate alternatives: `nomic-embed-text-v1.5` (8k ctx, multilingual, code-aware, ~150 MB Q8), `bge-small-en-v1.5` (512 ctx), `jina-code-v2`, `CodeRankEmbed`
- [ ] Model-swap path: config field `semantic_search.embedder_model`; re-embed on change with progress UI
- [ ] Retire hardcoded bundling of MiniLM; `models/` directory holds whichever is active
- [ ] Bundle `bge-reranker-v2-m3` GGUF (or similar small cross-encoder)
- [ ] Rerank stage in `search_semantic` / `search_hybrid`: bi-encoder top-K=50 → cross-encoder rerank → return top-10
- [ ] Config: `semantic_search.reranker_enabled`, `reranker_top_k`, `reranker_model`
- [ ] Catch2 perf test: reranker adds < 200 ms on 50 candidates on CPU
- [ ] Migration handling for existing workspaces (flag embedding dimension mismatch, offer re-index)
