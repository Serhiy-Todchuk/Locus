# S4.F — KV / Prompt Cache Preservation

## Problem

llama.cpp / LM Studio support prefix caching — but only when the system-prompt prefix is byte-stable across turns. Any per-turn change in `SystemPromptBuilder` output invalidates the cache, re-processing 2000+ tokens on every turn (10–30 s on a 26B model). Needs an audit.

## Tasks

- [ ] Audit `SystemPromptBuilder::build()`: identify all dynamic segments (timestamps, file counts, session IDs, last-indexed times)
- [ ] Reorder: static segments (base + LOCUS.md + tool schema) first; dynamic segments moved to first user turn or a separate "workspace state" message
- [ ] Verify cache hit with LM Studio verbose logging / llama.cpp `-v`
- [ ] Add `kv_cache_hits_est` metric to activity panel (based on prefix byte stability per turn)
- [ ] Document invariant in `system_prompt.h`: "prefix must be byte-stable across turns within a session"
