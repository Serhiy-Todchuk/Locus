# S4.B — Checkpoint & Undo

## Problem

No way to revert AI-made changes short of `git reset`. For a "transparency + user control" product, absence of undo is embarrassing. RooCode / Cline both have one-click turn revert.

## Tasks

- [ ] Pre-mutation snapshot: on any write/create/delete/edit tool success, copy prior file state to `.locus/checkpoints/<session_id>/<turn_id>/<relative_path>`
- [ ] `checkpoint.json` per turn: list of `(path, action, prev_hash)` for deterministic restore
- [ ] `ILocusCore::undo_turn(session_id, turn_id)` — restore all files from that turn's snapshot, emit activity event
- [ ] GC: keep last N=50 turns per session (configurable); prune on session close
- [ ] Skip large files (> 1 MB) with a warning — defer to git for those
- [ ] UI: "Undo last turn" button in chat footer; per-turn revert from activity panel
- [ ] Catch2 tests: snapshot/restore round-trip, GC eviction, size-limit skip
