# S4.C — Auto-Verify Feedback Loop

## Problem

Agent writes broken code, reports success. Aider's `lint_cmd`/`test_cmd` and Cline's terminal watching solve this. `run_command` already exists but nothing triggers it automatically — for a C++ project that means template/link errors reach the human instead of being caught by the agent's own loop.

## Tasks

- [ ] `.locus/config.json`: `verify.cmd` (string), `verify.trigger` (`on_edit` | `on_turn_end` | `manual`), `verify.timeout_ms`
- [ ] Post-mutation hook: run verify command after any file-mutating tool success
- [ ] Non-zero exit: inject `<verify_failed>` message with stdout/stderr tail (last N lines) so LLM can react in same turn
- [ ] Zero exit: log only, no context injection (token discipline)
- [ ] UI: verify status chip in footer (green check / red fail / spinner)
- [ ] Per-language default commands shipped: `cmake --build build`, `tsc --noEmit`, `cargo check`, `pytest -q`
- [ ] Catch2 tests: trigger fires on mutation, zero-exit silence, non-zero injection
