# S4.D — Plan Mode

## Problem

OpenCode, Cline, RooCode, Claude Code all have "propose plan → user approves plan → agent executes plan" workflow. Matches Locus's "user as architect" thesis exactly. Current step-by-step approval is too granular for multi-step work — user sees trees, not forest.

## Tasks

- [ ] Agent mode enum: `chat` (current), `plan` (propose steps, tool execution disabled), `execute` (tools allowed, plan as context)
- [ ] System-prompt variant for plan mode: "produce numbered plan, no tool calls"
- [ ] `propose_plan` tool (plan-mode only): `steps: [{description, tools_needed}]` — displayed for approval
- [ ] On plan approval: switch to execute mode; plan stays pinned in context
- [ ] UI: mode switcher (chat/plan/execute) in chat input area; plan panel with step checkboxes
- [ ] Per-step auto-approve: user can whitelist tools for pre-approved plan steps
- [ ] Catch2 tests: mode transitions, tool-blocking in plan mode
