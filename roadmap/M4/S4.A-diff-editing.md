# S4.A — Diff-Based Editing

## Problem

`WriteFileTool` overwrites whole files. On local LLMs (Gemma-27B, Qwen3, Llama3) this (a) corrupts files via drift / whitespace damage / silent function deletion, (b) costs 10–100× more tokens than needed, (c) feels 20× slower on mid-range GPUs. Every competitive agent uses diff-style edits (Aider `udiff`/`diff-fenced`, Cline/RooCode `replace_in_file`, Claude Code `Edit old_string/new_string`).

## Tasks

- [x] `edit_file` tool: `path`, `old_string`, `new_string`, optional `replace_all` — exact-match with uniqueness check (fail if `old_string` ambiguous)
- [x] Require prior `read_file` on the same path in the same session before editing (Claude Code pattern) — reduces hallucinated edits
- [x] Multi-edit variant: atomic list of `(old_string, new_string)` pairs on a single file
- [ ] Evaluate alternative format: unified-diff (`apply_patch`) — compare which local models produce cleaner output
- [x] Preview in approval panel: red/green diff via wxStyledTextCtrl markers, not raw text
- [x] Update system prompt guidance: edit > rewrite; `write_file` only for fresh files or full rewrites
- [x] Catch2 tests: uniqueness check, non-existent-match failure, indent preservation, multi-edit atomicity

## Implementation notes

- Exact-match engine lives in [src/tools/file_tools.cpp](../../src/tools/file_tools.cpp); `EditFileTool` and `MultiEditFileTool` share `apply_single_edit` / `replace_all_occurrences`. Writes go through a temp file + rename to stay atomic on crash.
- "Must read before edit" is enforced by a process-wide `ReadTracker` in [src/tools/shared.h](../../src/tools/shared.h) — `ReadFileTool` populates it on success, the edit tools fail loudly otherwise. Canonical paths, so workspace switches do not collide. Tests `clear()` between cases.
- Unified-diff (`apply_patch`) format is deferred — revisit when M4.J/M4.K let us A/B local models on patch-style output.
- Approval UI renders edit tools as a colored unified diff inside the existing Scintilla pane (`render_diff_view()` in [src/gui/tool_approval_panel.cpp](../../src/gui/tool_approval_panel.cpp)); `Modify` flips back to raw JSON, then back to diff on cancel.
