# S4.A — Diff-Based Editing

## Problem

`WriteFileTool` overwrites whole files. On local LLMs (Gemma-27B, Qwen3, Llama3) this (a) corrupts files via drift / whitespace damage / silent function deletion, (b) costs 10–100× more tokens than needed, (c) feels 20× slower on mid-range GPUs. Every competitive agent uses diff-style edits (Aider `udiff`/`diff-fenced`, Cline/RooCode `replace_in_file`, Claude Code `Edit old_string/new_string`).

## Tasks

- [ ] `edit_file` tool: `path`, `old_string`, `new_string`, optional `replace_all` — exact-match with uniqueness check (fail if `old_string` ambiguous)
- [ ] Require prior `read_file` on the same path in the same session before editing (Claude Code pattern) — reduces hallucinated edits
- [ ] Multi-edit variant: atomic list of `(old_string, new_string)` pairs on a single file
- [ ] Evaluate alternative format: unified-diff (`apply_patch`) — compare which local models produce cleaner output
- [ ] Preview in approval panel: red/green diff via wxStyledTextCtrl markers, not raw text
- [ ] Update system prompt guidance: edit > rewrite; `write_file` only for fresh files or full rewrites
- [ ] Catch2 tests: uniqueness check, non-existent-match failure, indent preservation, multi-edit atomicity
