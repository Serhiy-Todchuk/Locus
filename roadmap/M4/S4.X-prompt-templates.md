# S4.X -- Prompt Templates

## Problem

Repetitive workflows ("write me a standup from this week's diff", "produce a postmortem outline", "review this PR for security issues") get retyped every session. Pi solves this with prompt templates: Markdown files where typing `/<name>` expands the file's contents inline as the user message. We have slash commands that invoke *code*; we have nothing for slash commands that expand to *text*.

This is the cheapest, highest-value extension surface from the Pi comparison and does not conflict with the user-as-architect thesis -- templates are author-controlled Markdown, no surprise behavior.

## Tasks

- [ ] On-disk layout:
  - Project-local: `.locus/prompts/<name>.md`
  - Global: `%APPDATA%/Locus/prompts/<name>.md` (Windows), `~/.locus/prompts/<name>.md` (Unix)
  - Project-local wins on name collision; both directories are scanned at workspace open.
- [ ] `PromptTemplateRegistry` in [src/agent/](../../src/agent/): scans both directories, reads each `.md` file lazily (cache file content + mtime, re-read on change), exposes `list()` and `expand(name, args)`.
- [ ] Variable substitution: positional `{0}` `{1}` ... and named `{key}`. Caller passes `args = ["foo", "bar"]` and `kwargs = {key: "value"}` from the slash-command parser. Missing references render as the literal placeholder so the user sees what was unresolved. Keep templating intentionally minimal -- no conditionals, no loops; if a template needs logic it should be a real extension.
- [ ] Wire into `SlashCommandDispatcher` (S3.J): if a `/foo` token isn't a built-in command, look it up in `PromptTemplateRegistry`. On match, expand and inject as a user message into the conversation; on miss, surface the existing "unknown command" error with a suggestion list of nearby template names.
- [ ] `/help` lists available templates alongside built-in commands, grouped under "Templates (project)" and "Templates (global)".
- [ ] Tab-completion for `/` integrates template names (extends the existing `complete(prefix)` surface).
- [ ] Token-budget guard: warn when an expanded template + current conversation exceeds 50% of the model's context window. Cheap heuristic; better than letting the user accidentally torch a tight 8k context with a long template.
- [ ] Hot-reload: re-scan on `/reload` (new built-in slash command) and on workspace switch. mtime-based cache invalidation handles edits without a full rescan.
- [ ] Frontmatter (optional, for ergonomics, not required for v1):
  ```
  ---
  description: Generate a standup from this week's git activity
  args: [target_date]
  ---
  ```
  `description` shows in `/help`; `args` declares positional names so the dispatcher can validate.
- [ ] Catch2 tests in `tests/test_prompt_templates.cpp`:
  - registry scans both dirs; project wins on collision
  - positional + named substitution; missing key renders literal
  - frontmatter parsing (with and without)
  - mtime-based cache invalidation
  - dispatcher integration: `/foo arg1` round-trips to expanded user message
  - `/help` includes templates

## Implementation notes

- `PromptTemplateRegistry` lives in `src/agent/` next to `slash_commands.h/cpp` -- it is a peer of the slash-command system, not part of it. The dispatcher consults it as a fallback resolver.
- No execution context inside templates (no shell, no tool calls, no LLM passes). Pure text expansion. This is the security model: a template can phrase a request, but it cannot perform a request. The agent then chooses whether to call tools as it would for any user message.
- Distribution is deliberately out of scope for this stage. Users copy `.md` files in by hand or `git clone` a template repo into `.locus/prompts/`. The "package install" story is M5 [S5.7 -- Skills](../M5/S5.7-skills.md), which subsumes prompt templates as one component of a skill bundle.
- Frontmatter parser: minimal hand-rolled YAML-subset (just `key: value` and `key: [a, b, c]`). Don't pull in a YAML library for this; Markdown's existing dependencies stay clean.
- Naming: `/help` deliberately groups built-ins above templates so a malicious template named `compact` can't shadow `/compact` -- built-ins take precedence in the dispatcher.
