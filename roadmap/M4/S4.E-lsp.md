# S4.E — LSP Integration

## Problem

Tree-sitter gives a shallow symbol table. LSP gives real semantics: cross-TU go-to-def, find-references, rename-symbol, compiler diagnostics. For C++ (templates, overloads, namespaces) and TS, `clangd` / `tsserver` trounce Tree-sitter. Already on TODO.txt.

## Tasks

- [ ] `LspClient` class: spawn language server as child process, JSON-RPC over stdio
- [ ] Server registry in config: `lsp.servers: { cpp: "clangd", python: "pylsp", typescript: "tsserver" }`
- [ ] Lazy startup: spawn server on first request for a language; shut down on workspace close
- [ ] Tool: `goto_definition(path, line, col)` → list of `(path, line, col)`
- [ ] Tool: `find_references(path, line, col)` → list of `(path, line, col, snippet)`
- [ ] Tool: `get_diagnostics(path?)` → errors/warnings; can feed verify loop as a second signal
- [ ] Tool: `rename_symbol(path, line, col, new_name)` → preview all edits across files, single approval gate
- [ ] Tool: `hover_info(path, line, col)` → type info + docstring
- [ ] Catch2 tests with bundled clangd: go-to-def across TUs, find-refs accuracy
