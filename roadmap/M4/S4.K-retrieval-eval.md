# S4.K — Retrieval Evaluation Harness

## Problem

Zero way to measure if hybrid search gets better or worse when tweaked. Current retrieval ships on vibes.

## Tasks

- [ ] `tests/retrieval_eval/queries.json`: 50+ gold queries — `{query, expected_files[], expected_symbols[]}` against WS1 itself
- [ ] Evaluator binary: runs each query through `search_text`, `search_semantic`, `search_hybrid`; computes recall@K, MRR, nDCG
- [ ] CI target: run eval on every build; fail if recall@3 regresses > 5%
- [ ] Report format: markdown table committed to `tests/retrieval_eval/results.md` on each run
- [ ] Curate queries from real sessions (activity-log replay tool)
