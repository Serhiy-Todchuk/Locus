# S4.N — Tool-Call Robustness Across Model Families

## Problem

Current tool-call parsing assumes LM Studio's OpenAI-flavored JSON. Gemma-3's tool-use is shaky; Qwen3 uses `<tool_call>` XML natively; Llama3 needs prompt engineering. When the user swaps models (they will), tool calls silently break.

## Tasks

- [ ] Model-aware tool-call parser: OpenAI JSON (current), Qwen-style `<tool_call>`, Claude-style `<function_calls>`, Llama3 plain JSON blocks
- [ ] Heuristic detection: sniff first tool-call token to choose parser
- [ ] Per-model config override in `.locus/config.json`: `llm.tool_format: openai | qwen | xml | auto`
- [ ] System-prompt variants per model family: inject format-specific instructions
- [ ] Graceful fallback: if JSON parse fails mid-stream, attempt XML parse before giving up
- [ ] Catch2 tests: each format's parser against recorded fixtures
