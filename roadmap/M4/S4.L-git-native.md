# S4.L — Git-Native Features

## Problem

Aider's identity is git-native. `.gitignore` respect by indexer needs verification. Git is already the canonical undo; lean into it for tools.

## Tasks

- [ ] Verify: indexer respects `.gitignore` on top of user-configured excludes; if not, parse via libgit2 or pattern-match
- [ ] Tool: `git_status()` — ported/modified/staged (read-only, auto-approve)
- [ ] Tool: `git_diff(path?, staged?)` — unified diff output (read-only, auto-approve)
- [ ] Tool: `git_log(path?, n?)` — last N commits with author/message (read-only, auto-approve)
- [ ] Tool: `git_blame(path, line_start, line_end)` — who wrote this (read-only, auto-approve)
- [ ] Optional auto-commit-per-turn (Aider-style), behind `git.auto_commit` flag; commit message from LLM summary
- [ ] `.gitignore` watcher: re-read on change, re-apply to index
- [ ] Dependency: libgit2 via vcpkg (avoid shelling `git` for perf + portability)
