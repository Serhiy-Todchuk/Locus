# S4.G — MCP (Model Context Protocol) Client

## Problem

Every serious agent in 2026 speaks MCP: Claude Code, Cline, RooCode, OpenCode, Continue, Cursor. It's how users plug in Jira, Postgres, Playwright, Figma, internal tools. Locus has 12 hardcoded tools and zero plugin story (community-plugin item is deferred to post-M3 in the old plan — that's wrong; MCP is the plugin story).

## Tasks

- [ ] `McpClient` class: spawn MCP stdio server as child process; handshake via JSON-RPC
- [ ] `.locus/mcp.json` + global `%APPDATA%/Locus/mcp.json`: list of servers with `command`, `args`, `env`
- [ ] Discover tools via `tools/list`; register into `ToolRegistry` with namespaced names (`mcp:servername:toolname`)
- [ ] Proxy tool calls: `execute()` on MCP tool → JSON-RPC call → result back to agent
- [ ] Per-server approval policy default: `always` (MCP tools untrusted until user opts in)
- [ ] UI: MCP server panel in settings (running/stopped, tool count, logs tail)
- [ ] Graceful degradation: server crash → tools unregistered with warning event
- [ ] Streamable-HTTP MCP transport as follow-up (after stdio works)
- [ ] Catch2 tests with a bundled mock MCP server
