# S4.I — Background / Long-Running Commands

## Problem

`run_command` blocks with 30s timeout. Cannot start dev server, run long builds (cmake 2min+), tail logs. Claude Code's `run_in_background` + `Monitor` solves this.

## Tasks

- [ ] `run_command_bg(command)` → returns `process_id`, stays alive across turns
- [ ] `read_process_output(pid, since_offset?)` → stream bytes since last read
- [ ] `stop_process(pid)` → terminate, capture final exit code
- [ ] `list_processes()` → active background processes `(pid, command, started_at, status)`
- [ ] Process registry owned by Core; persists across sessions, terminated on workspace close
- [ ] Auto-truncate output buffer per process (ring buffer, configurable cap)
- [ ] UI: background process panel (tray badge count, kill buttons)
