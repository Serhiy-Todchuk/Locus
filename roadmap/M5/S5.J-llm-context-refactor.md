# S5.J -- LLMContext Class Refactor

## Problem

`AgentCore` currently owns 14 members. **Nine of them describe what the LLM sees** -- conversation history, the assembled system prompt, the token budget, file-change tracking, attached context, checkpoints, the saved-session manager. The other five are agent-loop machinery (the loop itself, the dispatcher, slash commands, metrics, the agent thread).

This split is exactly what S3.A meant to solve when it extracted `AgentLoop` / `ToolDispatcher` / `ActivityLog` / `ContextBudget` from the original 850-LOC `AgentCore`. The slot labelled "ConversationHistory" has quietly absorbed everything LLM-view-shaped since then. With M5 about to add per-message token estimates ([S5.D](S5.D-reserve-and-token-xray.md)), pin states ([S5.E](S5.E-pin-and-tag.md)), composable compaction layers ([S5.F](S5.F-compaction-v2.md)), edit / delete / rewind operations ([S5.H](S5.H-edit-delete-branch.md)), and per-tab cloneability ([S5.I](S5.I-tabs-and-sessions.md)), the class is on the verge of god-class again.

Pay it down once, before the feature work, the same way S3 paid down for M4.

## What this stage does

Introduce **`LLMContext`** -- a single owner for the conversation's full LLM-facing state. AgentCore composes it instead of orchestrating its parts directly.

Per the design discussion:
- Name **`LLMContext`** (not `Conversation` or `ConversationContext`): scopes the abstraction to *what the LLM sees*, distinct from the existing `AttachedContext` and `IWorkspaceServices`.
- **Scope (b)**: not just the field move -- also extract the system-prompt assembly into its own immutable type (`SystemPromptAssembly`). [S4.F](../M4/S4.F-kv-cache.md)'s byte-stable invariant becomes a property of the type, not a discipline `AgentCore` has to remember.
- **Refactor all touching tests** -- the suite is the safety net for this refactor and most tests get *simpler* by constructing an `LLMContext` directly without an agent thread.

## The new shape

### `src/agent/llm_context.h/cpp`

```cpp
class LLMContext {
public:
    LLMContext(IWorkspaceServices&, ILLMClient&, IToolRegistry&,
               LLMContextConfig);

    // -- Read surface ---------------------------------------------------
    ConversationSnapshot snapshot() const;        // immutable view
    int  current_tokens()  const;                 // server-reported or estimated
    int  effective_limit() const;                 // limit - reserve_tokens (S5.D)
    int  reserve_tokens()  const;                 // S5.D
    PromptBreakdown system_prompt_breakdown() const;  // S5.G stacked-bar source

    const SystemPromptAssembly& system_prompt() const;  // immutable

    // -- Mutations (owner-thread-only) ----------------------------------
    MessageId add_user_message(std::string, AttributedSource = User);
    MessageId add_assistant_message(...);
    MessageId add_tool_call(ToolCall);
    MessageId add_tool_result(MessageId call_id, ToolResult);

    // S5.E
    void     set_pin_state(MessageId, PinState);
    PinState pin_state(MessageId) const;

    // S5.H
    void edit_message(MessageId, std::string);
    void delete_message(MessageId);
    void rewind_to(MessageId);                    // archives the dropped lineage

    // S5.D enforcement
    bool would_breach_reserve(int incoming_estimate) const;

    // S5.F
    CompactionPreview preview_compaction(LayerSelection) const;
    CompactionResult  run_compaction(LayerSelection);
    std::vector<ArchiveInfo> list_archives() const;
    void restore_archive(ArchiveId);

    // Persistence (S5.I)
    SessionId save_session();
    static std::unique_ptr<LLMContext> load_session(
        SessionId, IWorkspaceServices&, ILLMClient&, IToolRegistry&);

    // S4.F canary
    size_t system_prompt_hash() const;

private:
    SystemPromptAssembly  prompt_;          // immutable post-construction
    ConversationHistory   history_;
    ContextBudget         budget_;
    FileChangeTracker     file_tracker_;
    CompactionPipeline    compaction_;      // empty stub in v1; filled in S5.F
    HistoryArchive        archive_;         // empty stub in v1; filled in S5.F/S5.H
    CheckpointStore       checkpoints_;
    SessionManager&       sessions_;        // shared, not owned
    AttachedContextSlot   attached_;        // S4.F migration target
    LLMContextConfig      config_;
};
```

### `src/agent/system_prompt_assembly.h/cpp`

`SystemPromptBuilder::build()` becomes a private helper of a new immutable type:

```cpp
class SystemPromptAssembly {
public:
    static SystemPromptAssembly build(const WorkspaceMetadata&,
                                       const std::string& locus_md,
                                       const IToolRegistry&,
                                       ToolFormat);

    // Read-only access
    const std::string& full_text() const;
    const std::string& base_text() const;
    int  total_tokens()       const;
    int  base_tokens()        const;
    int  locus_md_tokens()    const;
    int  metadata_tokens()    const;
    int  manifest_tokens()    const;
    int  format_addendum_tokens() const;
    size_t hash()             const;

private:
    SystemPromptAssembly() = default;        // build() is the only constructor
    // ... fields populated by build()
};
```

The "byte-stable across the session" invariant is now a *type-level* guarantee: the only way to get a `SystemPromptAssembly` is via `build()`; all read methods are `const`. There is no setter. [S4.F](../M4/S4.F-kv-cache.md)'s debug-only hash canary still fires (now `LLMContext::system_prompt_hash()`), but the structural protection is the type, not the discipline.

### `AgentCore` after

Slimmed back to the agent loop runner. Holds a reference (or `unique_ptr` for now -- TBD during implementation) to `LLMContext`:

```cpp
class AgentCore {
public:
    AgentCore(LLMContext&, ILLMClient&, IToolRegistry&, /* config */);
    void start();
    void send_message(std::string);
    // ... loop control
private:
    LLMContext&                             ctx_;
    std::unique_ptr<AgentLoop>              loop_;
    std::unique_ptr<ToolDispatcher>         dispatcher_;
    std::unique_ptr<SlashCommandDispatcher> slash_;
    std::unique_ptr<MetricsAggregator>      metrics_;
    std::thread                             agent_thread_;
    std::queue<std::string>                 message_queue_;
    // ... lifecycle bits
};
```

`ConversationOwnerScope` discipline preserved -- `LLMContext` mutations are gated to the agent thread during turn execution; the existing RAII gate moves to wrap `LLMContext` writes instead of `ConversationHistory` writes directly.

## Acceptance criteria

This is a pure refactor; the bar is unchanged behaviour.

- All existing Catch2 tests (post-migration to the new shape) pass.
- `locus.exe d:/Projects/AICodeAss -verbose` -- a manual session against WS1 produces the same activity log shape as before, modulo logger prefix changes if any.
- Saved sessions written by the previous AgentCore-based code load correctly under the new LLMContext-based code.
- The S4.F debug-only hash canary (when in debug builds) reports zero unexpected system-prompt mutations across a normal session.

## Out of scope (deliberately)

- **No new behaviour.** No reserve enforcement (that's S5.D), no pin states (S5.E), no compaction layers (S5.F), no chat restructure (S5.G), no edit/delete/rewind (S5.H), no tabs (S5.I). Those land on the LLMContext surface this stage establishes.
- **No interface for `LLMContext` polymorphism (no `ILLMContext`).** Concrete class only. If we ever need a remote-context surface (Crow over LAN streams a `ConversationSnapshot`), introduce the interface then.
- **No `SessionManager` ownership change.** Stays at `AgentCore` level for now; `LLMContext` accesses it via reference. Migrating it is its own decision and not blocking anything.

## Implementation notes

- The refactor lands as one stage rather than incremental moves because the boundary it draws (LLM-view state vs agent-loop machinery) is a structural one. Doing it in pieces would mean the boundary is wrong half the time and tests reflect both old and new shapes simultaneously. One stage, one mental model.
- The `ConversationOwnerScope` RAII gate ([S3.I](../M3/S3.I-threading-model.md)) keeps the same role: writes to `LLMContext` mutating methods must hold the scope. The scope is now bound to `LLMContext` rather than `ConversationHistory`. Threading invariant unchanged.
- `SystemPromptAssembly`'s immutability is the structural fix for the [S4.F](../M4/S4.F-kv-cache.md) byte-stability invariant. Today it lives as a discipline ("don't recompute `base_system_prompt_`"); after this stage it lives in the type system (no setter exists). This is the single biggest correctness improvement of the refactor.
- The `LLMContextConfig` struct gathers the soft / hard thresholds, reserve tokens (S5.D's input), compaction layer defaults (S5.F's input). It's plumbed in at construction. Workspace-config changes that affect these values trigger a Context rebuild on the next message (in line with how config changes already work today).
- The compaction-pipeline and history-archive stubs in v1 contain enough surface for AgentCore not to crash but produce no-op results. S5.F fills them in. Same pattern as S3.A's stubbed-then-filled hooks.
- Tabs ([S5.I](S5.I-tabs-and-sessions.md)) become structurally simple after this lands: a tab is `{ LLMContext, AgentCore }` -- two members, both already supporting independent instances. The current shape (AgentCore owning everything) would have made tabs a multi-week refactor; this stage is the prerequisite.

## Manifest cost

**Zero new tools.** Pure internal refactor. The agent's tool surface is unchanged.
